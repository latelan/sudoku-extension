数独
====

有以下几个问题：

1. 输入转换
   输入为'e'或'o': 转换为偶数或基数添加即可.

2. 怎么处理相同字母的格子
   在remove和resume时，同时删除或增加相同的值所在的行.

3. 怎么使得得分最高
  

优化：
----
1. 将已知节点优先加入DL中,具体顺序
	已知 > 字母e或o > 相同字母 > 数字0

在Ubuntu1204上测试结果：
未优化顺序前
	$time ./driver.sh > 1
某一次所用时间，基本都在这个左右
	real    0m0.221s
	user    0m0.193s
	sys     0m0.027s

优化顺序后
	$time ./driver.sh > 1
某一次所用时间，没超过0.3s
	real    0m0.208s
	user    0m0.179s
	sys     0m0.027s
看来是有效果的。

2.添加已知节点时，设置该列的元素个数cnt[Col]为-1，后面添加未知节点时，
  若该列元素个数为-1，则不添加。

在Ubuntu1204上测试时，发现效果不明显，但理论上是优化了。


猜想实验：
--------
猜想：在添加每个格子可能数字时，默认顺序是1-9，反过来可能得到更高的分。
经实验，是错误的。分数反而更低。
发现：有一些结果分数要高于默认，最终方案可采用两种方式计算，分数取其高者
